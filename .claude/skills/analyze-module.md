# Skill: Analyze Module

## Goal

Understand an unfamiliar module quickly and produce a practical implementation plan.

## Flow

1. Map entry points and callers.
2. Identify data contracts (inputs/outputs/events).
3. Identify dependencies (state, API, storage, CSS, docs).
4. List risks and hidden coupling.
5. Propose change plan with smallest safe steps.

## Output template

- Module purpose
- Key files
- Data flow
- Risk points
- Proposed implementation steps
- Validation plan
